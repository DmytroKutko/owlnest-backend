# Handoff Contracts

Use stable IDs and Markdown/YAML-like sections. Omit no required field; use `NOT_APPLICABLE` with a reason. Every handoff includes `MASTER_REQUEST`, scope, evidence paths, assumptions, unresolved questions, non-goals, authoring role, timestamp, and status.

## 18. Code-review finding

Use the common finding plus `SPRING_LIFECYCLE_OR_CORRECTNESS_IMPACT`, `CALLERS_AFFECTED`, and `VERIFICATION_COMMAND`.

## 19. Migration-review finding

Use the common finding plus `MIGRATION_FILE_VERSION`, `LOCK_OR_REWRITE_RISK`, `ROLLING_COMPATIBILITY`, `DATA_LOSS_RISK`, `FORWARD_FIX`, and `POSTGRES_VALIDATION`.

## 20. API QA finding

Use the common finding plus `REQUEST`, `EXPECTED_STATUS_BODY`, `ACTUAL_STATUS_BODY`, `AUTH_CONTEXT`, `POSTGRES_EFFECT`, `REDIS_EFFECT`, and `OPENAPI_MISMATCH`.

## 21. Regression QA finding

Use the common finding plus `BASELINE_RESULT`, `CURRENT_RESULT`, `REGRESSION_SCOPE`, `TEST_OR_MANUAL_CASE`, and `INTRODUCED_FAILURE_EVIDENCE`.
